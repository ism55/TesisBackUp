Como recomendación se propone mejorar este prototipo añadiendo conectividad a internet, en el cuál se puede hacer uso de recursos en la nube para implementar una aplicación web mucho más eficiente.

Por otra parte, se pueden implementar otros métodos de control al sistema controlador. Esto es utilizando este prototipo que ofrece comunicación remota para adquirir información sobre el movimiento del brazo y recopilarlas para realizar control mediante algoritmos de redes neuronales o aprendizaje de máquinas.

También puede hacerse uso de software de realidad aumentada para generar los comandos de movimiento a través de una aplicación que monitorea vía cámaras web el brazo robot y a su vez transmitir esta comunicación al módulo prototipo.

Una mejora de este prototipo podría realizarse al implementar la comunicación bidireccional entre el brazo robot y el cliente. Ya que con esto puede utilizarse la información para realizar mejores controladores.

En cuanto a la librería de jQuery, puede utilizarse el particionamiento de la memoria estática del microcontrolador para guardar en la memoria del mismo, imágenes con mejor resolución y las librerías de mayor interés, como lo es la jQuery.

Se propone implementar la solución realizada en este trabajo a través de otros dispositivos que posean conectividad a través de Wi-Fi, para de esta forma comparar la eficiencia entre estos dispositivos para lograr un manejo remoto óptimo.
