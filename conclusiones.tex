Luego de un estudio sobre las técnicas y metodologías comúnmente utilizadas para el manejo de brazos manipuladores, se determinaron las acciones del sistema de control del brazo manipulador MA2000. Con esta información se seleccionó el dispositivo ESP8266 ya que se ajustó a los requerimientos del diseño. Se seleccionó la comunicación vía Wi-Fi como modo de acceso remoto para el prototipo propuesto.

Se logró diseñar un prototipo de sistema de acceso remoto al controlador del brazo manipulador MA2000 ubicado en el LCID que permitió el movimiento de este sin la presencia del software original ni conexiones alámbricas diferentes al puerto serial del microcontrolador.

El brazo robot puede manejarse a través del software Robocom desarrollado en \cite{juan1} comunicándose a través de un convertidor USB-serial conectado al puerto serial de un microcontrolador Microchip dsPIC30F3011 sin ninguna alteración. No obstante el prototipo realizado en este proyecto agregó una mejora en el manejo del brazo robot, permitiendo que este pueda utilizarse sin el software, logrando acceder al sistema de movimiento mediante el uso de una página web sin usar el módulo convertidor USB-Serial FTDI232.

La utilización del microcontrolador ESP8266 para establecer comunicación vía Wi-Fi a través de un servidor alojado en el sistema embebido, demostró ajustarse a las necesidades del proyecto, que no requería un hardware de gran nivel para alcanzar el objetivo. Este microcontrolador ayudó a establecer una comunicación en tiempo real en la generación de nuevas tramas de movimiento.


Se desarrolló un documento digital que sirvió como Manual para la configuración y utilización del módulo de acceso remoto y como recurso para la validación del funcionamiento del sistema diseñado. Este manual puede accederse en línea a través de la dirección ism55.github.io/ManualTesis y en el se encuentran los videos de funcionamiento, solución a problemas y la utilización de las versiones web y Android de la interfaz de usuario.
